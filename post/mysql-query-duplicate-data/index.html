查询重复数据的记录数

有时候，因为一些“意外”会在在数据库里生成一些冗余重复的数据，为了避免这些脏数据对项目后续的数据交互使用有影响，我们需要先将其查找出来，再进行分析情况进而开展更进一步的处理。

SELECT <重复数据字段名>, COUNT(*) AS COUNT FROM `<待查询表名>` GROUP BY <重复数据字段名> HAVING COUNT > 1;

有一说一，这个查询是我在网上找的，不过我们来理一理为啥可以这样写。

假定我们需要看 user_info 表中 user_id 的重复情况：每个 user_id 对应的值和其重复的条数，毕竟不是所有的 user_id 的值都重复了。

SELECT user_id, COUNT(user_id) FROM user_info;

注意：

  • COUNT 函数，有两种使用方式，会有不同的结果：
    • COUNT(*)：对表中行的数目进行计数，不管列的值是不是 NULL。
    • COUNT(列名)：对特定的列进行计数，会忽略掉该列为 NULL 的行。

不过，这个查询语句看似没毛病，但实际是有问题的。

因为在无 GROUP BY 的 SELECT 查询中，被 SELECT 的只能要么都是聚合函数，要么都不是。否则就会出问题，要么报错，要么返回个诡异的结果。

比如，在 MySQL 8.0.11 会报错：

SELECT user_id, COUNT(user_id) FROM user_info
> 1140 - In aggregated query without GROUP BY, expression #1 of SELECT list contains nonaggregated column 'demo_utf8mb4_unspecified_sort.user_info.user_id'; this is incompatible with sql_mode=only_full_group_by
> 时间: 0s

而在 MySQL 5.7.30 中不会报错，但显示的结果很诡异：结果集只有一条记录！且 user_id 列下的值是数据库记录对应列第一条的值，COUNT(user_id) 列下的值是显示了整个数据库指定列下除了 NULL 的总条数。这个结果显然不符合预期的。

顺便也提下，如何使用 SQL 语句查询当前 MySQL 版本：

SELECT VERSION();
